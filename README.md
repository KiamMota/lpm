# lpm — Lazy Plugin Manager

**The CLI for lazy.nvim.** manage lazy.nvim plugins from your terminal.

![lpm fetch](assets/demo-lpm-go.gif)

---

## Commands

| Command | Description |
|---|---|
| `lpm install <github_url>` | Download and create plugin config |
| `lpm remove <plugin>` | Delete plugin config file |
| `lpm able <plugin>` | Enable/disable a plugin (toggle) |
| `lpm cat <plugin>` | Print plugin config to terminal |
| `lpm list` | List installed plugins |
| `lpm --version` / `-v` | Show version |
| `lpm --fetch` / `-f`| Fetch Lpm |

> [!NOTE]
> All commands that take a plugin name (`remove`, `able`, `cat`, `edit`) use robust fuzzy matching (you never need to type the exact plugin name.)

# Installation

## Homebrew (macOS/Linux)
```bash
brew tap KiamMota/lpm
brew install lpm
```

## Fast Installation (Linux)
``` bash
curl -fsSL https://raw.githubusercontent.com/KiamMota/lpm/main/linux_install.sh | sh
```

## From source

Requirements: [Go 1.21+](https://go.dev/dl/)
```bash
git clone https://github.com/KiamMota/lpm
cd lpm
go install ./cmd/lpm
```


## How it works

`lpm` creates and manages `.lua` files in your lazy.nvim plugins folder. lazy.nvim picks them up automatically — no manual config needed.

```
lpm install → creates .lua file → lazy.nvim downloads the plugin
lpm remove  → deletes .lua file → lazy.nvim uninstalls on next sync
lpm able    → toggles the plugin on or off without deleting
```

Each plugin gets its own file:

```lua
-- automatically generated by lpm
return {
  'nvim-tree/nvim-tree.lua',
  dependencies = { 'nvim-web-devicons' },
}
```

If a curated config exists in the [lpm plugin registry](https://github.com/KiamMota/lpm/tree/main/plugins_config), it will be used instead of the generic template.

After any change, open Neovim and run `:Lazy sync` to apply.
