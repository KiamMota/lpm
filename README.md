# lazy.nvim-cli

Manage your Neovim plugins without opening the editor.

---

## Installation

```bash
cmake -B build && cmake --build build
sudo cmake --install build
```

---

## Commands

### Install a plugin

```bash
lazynvimcli install <github_url>
```

Creates a `.lua` file in `~/.config/nvim/lua/plugins/` ready for lazy.nvim to pick up.

```bash
lazynvimcli install https://github.com/nvim-telescope/telescope.nvim
# → creates ~/.config/nvim/lua/plugins/telescope.nvim.lua
```

---

### List installed plugins

```bash
lazynvimcli list
```

Lists all `.lua` files in your plugins folder.

---

### Remove a plugin

```bash
lazynvimcli remove <plugin-name>
```

Deletes the corresponding `.lua` file. lazy.nvim will uninstall it on the next sync.

```bash
lazynvimcli remove telescope.nvim
```

---

## How it works

Each plugin gets its own `.lua` file:

```lua
-- file automatically generated by lazy.nvim cli.
return {
  'nvim-telescope/telescope.nvim',
  dependencies = { 'nvim-treesitter/nvim-treesitter' },
}
```

Open Neovim and run `:Lazy sync` to apply changes.

---

## Requirements

- Neovim with [lazy.nvim](https://github.com/folke/lazy.nvim) configured
- Plugins folder at `~/.config/nvim/lua/plugins/` (Linux/Mac) or `%LOCALAPPDATA%\nvim\lua\plugins\` (Windows)
